¿Qué pasa cuando el cliente rechaza el presupuesto?

En el flujo actual existe el estado "Rechazado". ¿Lo mantenemos en el flujo simplificado?

Flujo opción C (confirmado)
📥 Ingresado → 🚚 En Taller o 🔧 En Reparación → 📞 Esp. Aprobación → 🎉 Listo p/Retiro → ✅ Entregado
A

Sí, mantener "Rechazado"

📞 Esp. Aprobación → ✗ Rechazado → 🎉 Listo p/Retiro

El cliente rechaza el presupuesto → se devuelve el artículo sin reparar. Útil para trackear cuántos presupuestos se rechazan y por qué.

B

No, ir directo a "Listo para Retiro"

📞 Esp. Aprobación → 🎉 Listo p/Retiro (sin reparar)

Si rechazan, directamente se mueve a "Listo para Retiro" — el artículo espera que lo vengan a buscar. Menos estados, más simple. El rechazo queda en las notas o en el historial.
